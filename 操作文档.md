# 宝信 API 测试管理平台 - 操作文档

## 一、平台概述

宝信 API 测试管理平台是基于 Flask 框架开发的接口自动化测试工具，提供接口管理、用例编排、测试执行、代码生成等一站式测试能力。

### 技术架构

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML/CSS/JavaScript | 原生前端，无需额外依赖 |
| 后端 | Flask 2.x | Python Web 框架 |
| 数据存储 | JSON 文件 | 轻量级数据持久化 |
| 测试执行 | requests 库 | HTTP 请求调用 |

### 项目结构

```
baoxinApiTest/
├── app.py              # 应用入口
├── configs/            # 配置文件
│   ├── config.py       # 环境配置、业务基础数据
│   └── configs.json    # 认证信息
├── data/               # 运行数据
│   ├── apis.json       # 接口定义
│   ├── modules.json    # 模块定义
│   ├── test_cases.json # 测试用例
│   ├── test_results.json # 测试结果
│   └── env_config.json # 环境配置
├── src/                # 核心代码
│   ├── models/         # 数据模型
│   ├── services/       # 业务服务
│   └── utils/          # 工具函数
└── templates/          # 前端页面
    ├── index.html      # 接口管理首页
    ├── cases.html      # 用例管理页面
    ├── runner.html     # 测试执行页面
    └── generator.html  # 代码生成页面
```

## 二、快速开始

### 2.1 环境要求

- Python 3.8+
- 依赖包：`flask`, `requests`, `lxml`

### 2.2 启动服务

```bash
# 进入项目目录
cd /Users/cengqi/Desktop/study/baoxinApiTest

# 安装依赖（首次）
pip install flask requests lxml

# 启动服务（默认端口 8888）
python3 app.py
```

### 2.3 访问地址

- 本地访问：http://localhost:8888
- 局域网访问：http://{IP}:8888

## 三、接口管理

### 3.1 页面入口

左侧导航栏点击「接口管理」，进入接口列表页面。

### 3.2 接口列表

接口列表展示所有已配置的接口，包含以下信息：

| 字段 | 说明 |
|------|------|
| 方法 | HTTP 请求方法（GET/POST/PUT/DELETE） |
| 接口名称 | 接口的中文名称 |
| Python函数 | 接口对应的函数名 |
| URL路径 | 接口的路径地址 |
| 所属模块 | 接口所属的业务模块 |

### 3.3 新增接口

点击「+新增接口」按钮，弹出新增弹窗，填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 接口名称 | 是 | 接口的中文标识 |
| Python函数名 | 是 | 调用时的函数名称 |
| 请求方法 | 是 | GET/POST/PUT/DELETE |
| URL路径 | 是 | 接口路径，如 `/api/contract/list` |
| 所属模块 | 是 | 选择业务模块 |
| 接口描述 | 否 | 接口功能说明 |
| 请求头 | 否 | JSON 格式，如 `{"Content-Type": "application/json"}` |
| 请求参数 | 否 | JSON 格式，定义接口变量 |
| 接口变量 | 否 | 接口依赖的变量列表 |

**接口变量配置示例：**

```json
{
  "cgName": "采购合同编号",
  "supplierName": "供应商名称"
}
```

### 3.4 编辑接口

点击接口列表的「编辑」按钮，修改接口信息。修改后，引用该接口的用例将自动使用最新内容。

### 3.5 删除接口

点击接口列表的「删除」按钮，确认后删除接口。**注意：删除前请确保没有用例引用该接口。**

### 3.6 测试接口

点击接口列表的「测试」按钮，验证接口是否可用。

**测试规则：**
- 如果接口配置了接口变量且值为空，点击测试会弹出提示「请输入接口变量」
- 测试结果会展示请求 URL、请求参数和响应内容
- 单独测试接口不会更新 `data.json` 缓存文件

## 四、用例管理

### 4.1 页面入口

左侧导航栏点击「用例管理」，进入用例列表页面。

### 4.2 用例列表

用例列表展示所有已配置的测试用例，支持按模块筛选和搜索。

### 4.3 新增用例

点击「+新增用例」按钮，弹出新增弹窗，填写以下信息：

#### 4.3.1 基本信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 用例名称 | 是 | 用例的中文标识 |
| 所属模块 | 是 | 选择业务模块 |
| 用例描述 | 否 | 用例功能说明 |

#### 4.3.2 临时变量

在「临时变量」区域配置用例级别的临时变量，格式为 JSON：

```json
{
  "name": "zq-采购-0702-001+{{random}}",
  "supplier": "供应商A"
}
```

**变量支持：**
- `{{random}}` - 生成随机数
- `{{timestamp}}` - 当前时间戳
- `{{uuid}}` - 生成 UUID

#### 4.3.3 测试步骤

点击「+添加步骤」，配置测试步骤：

1. **选择接口**：使用模糊查询下拉框选择接口
2. **接口返回字段**：配置需要提取的响应字段（可在后续步骤引用）
3. **变量参数**：设置接口变量的值

**变量参数配置规则：**

| 配置方式 | 示例 | 说明 |
|----------|------|------|
| 直接值 | `12345` | 固定值 |
| 引用临时变量 | `{{name}}` | 引用临时变量中的 name 值 |
| 引用接口返回 | `{{res}}` | 引用上一步接口返回值 |

#### 4.3.4 断言配置

点击「+添加断言」，配置断言规则：

| 字段 | 说明 |
|------|------|
| 左值 | 响应字段路径，如 `data.code` |
| 操作符 | 等于、不等于、包含等 |
| 右值 | 期望值，可引用变量，如 `{{cgName}}` |

### 4.4 执行用例

点击用例列表的「执行」按钮，执行测试用例。

**执行规则：**
- 用例执行不会修改 `data.json` 中的缓存数据
- 临时变量会在执行前解析并赋值给对应变量参数
- 执行结果会展示每个步骤的 URL、请求参数和响应内容

## 五、测试执行

### 5.1 页面入口

左侧导航栏点击「测试执行」，进入测试执行页面。

### 5.2 一键执行基础数据

点击「一键执行基础数据」按钮，执行所有基础数据接口。

**执行规则：**
- 此操作会更新 `data.json` 缓存文件
- 执行结果会保存到测试报告中

### 5.3 模块执行

选择业务模块，点击「执行模块」按钮，执行该模块下的所有接口。

## 六、代码生成

### 6.1 页面入口

左侧导航栏点击「代码生成」，进入代码生成页面。

### 6.2 生成测试代码

1. 选择需要生成代码的接口（可多选）
2. 点击「生成代码」按钮
3. 复制生成的 Python 测试代码

**生成的代码格式：**

```python
import unittest

class TestAPIs(unittest.TestCase):
    def setUp(self):
        """测试前准备"""
        pass
    
    def test_get_contract_info(self):
        """获取采购合同id信息"""
        result = get_contract_info()
        self.assertEqual(result['status'], 'success')
```

## 八、变量替换规则

### 8.1 变量类型

| 变量类型 | 格式 | 示例 |
|----------|------|------|
| 临时变量 | `{{name}}` | 引用用例临时变量中的 name 值 |
| 接口变量 | `{{cgName}}` | 引用接口配置的变量 |
| 系统变量 | `{{random}}` | 生成随机数 |
| 缓存变量 | `{{key}}` | 引用 data.json 中的缓存数据 |

### 8.2 变量优先级

1. 临时变量（用例级别）
2. 接口变量（接口级别）
3. 缓存变量（全局级别）

### 8.3 变量解析流程

```
执行用例
    ↓
解析临时变量（description字段）
    ↓
遍历测试步骤
    ↓
替换变量参数中的 {{变量名}}
    ↓
替换断言中的 {{变量名}}
    ↓
执行接口请求
    ↓
提取响应字段到缓存
    ↓
进入下一步骤
```

## 九、缓存机制

### 9.1 缓存文件

缓存数据存储在 `configs/data.json` 文件中，包含以下结构：

```json
{
  "datas": {
    "contractNo": "HT20240702001",
    "supplierName": "供应商A"
  },
  "headers": {
    "token": "xxxx"
  }
}
```

### 9.2 缓存更新规则

| 操作 | 是否更新缓存 |
|------|-------------|
| 一键执行基础数据 | 是 |
| 单独测试接口 | 否 |
| 执行用例 | 否 |
| 模块执行 | 否 |

## 十、常见问题

### 10.1 接口测试时提示「请输入接口变量」

**原因**：接口配置了接口变量，但变量值为空。

**解决方案**：在接口编辑页面填写接口变量的值，或在测试弹窗中输入变量值。

### 10.2 变量替换后值为 undefined

**原因**：引用的变量不存在或未正确解析。

**解决方案**：
- 检查临时变量是否正确配置
- 检查变量名是否拼写正确
- 确保变量在使用前已被定义

### 10.3 用例执行后缓存数据被清空

**原因**：之前版本的代码在执行用例时调用了 `clear_datas()`。

**解决方案**：当前版本已修复此问题，执行用例不会修改缓存数据。

### 10.4 测试结果弹窗无法关闭

**原因**：测试结果弹窗点击空白处可关闭，新增/修改弹窗点击空白处不能关闭。

**解决方案**：点击测试结果弹窗的空白区域即可关闭。

### 10.5 端口被占用

**原因**：8888 端口已被其他进程占用。

**解决方案**：

```bash
# 查找占用端口的进程
lsof -ti:8888

# 杀掉进程
lsof -ti:8888 | xargs -r kill -9

# 重新启动服务
python3 app.py
```
